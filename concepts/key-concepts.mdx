These are the terms the rest of the docs lean on. Learn these and the other pages read faster. For the full A to Z list, see the [Glossary](/concepts/glossary).

## App

Short for **Agentic Application**. The unit of deployment on Aomi. An App is the isolated environment an agent runs in, and it bundles four things: the tools it can call, the preamble that shapes its behavior, the model it runs on, and a scoped API key. Switching Apps changes the agent entirely. See [What is Aomi?](/concepts/what-is-aomi).

## Tool

A typed function the model can call during a conversation. A tool wraps an API, an onchain action, or a data source. Each tool has a name, a description, and typed input parameters, so the shape of every call is checked at build time. See [Custom Tools](/guides/custom-tools).

## Namespace

A set of host tools an App asks for. The default namespace is `evm-core`, which gives an App the EVM read and write primitives: read onchain state, stage and simulate a transaction, and request a wallet signature. The host injects the namespace's tools alongside the App's own. See [Architecture](/concepts/architecture).

## simulate

To run a transaction on a fork of the live network before it reaches the user's wallet. The user sees exact token changes, gas, and contract calls, then approves or rejects. This is the core safety step. See [How It Works](/concepts/how-it-works).

## non-custodial

A model where Aomi never holds keys or funds. Every transaction is signed locally in the user's wallet after the simulate step. The private key never leaves the user's device. See [Non-Custodial Automation](/concepts/non-custodial-wallets).

## Runtime

The hosted execution environment. It manages sessions, runs the model loop, runs the simulation pipeline, and streams responses. You ship an App and the Runtime runs it. See [Architecture](/concepts/architecture).

## widget

The `<AomiFrame />` chat component, installed into your project. It gives you a ready-made chat interface and the wallet UI. See [Widget Installation](/guides/widget-installation).

## headless library

The `@aomi-labs/react` package. It provides the runtime logic, state, and hooks with no UI opinions, so you can build a fully custom interface. The widget is built on top of it. See [Headless Library](/guides/headless-library).
