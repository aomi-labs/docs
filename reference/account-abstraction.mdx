title: Account Abstraction
description: How Aomi signs through account abstraction by default, with paymaster gas sponsorship, batched operations, and automatic EOA fallback.
---

Aomi signs transactions through account abstraction by default. When account abstraction is not available, it falls back to a direct EOA signature, so signing always completes either way.

## What you get

Signing through account abstraction unlocks two things on top of a plain wallet signature:

- **Gas sponsorship through a paymaster.** A paymaster covers the gas, so a user does not need to hold the gas token to transact.
- **Batched operations.** Several actions are bundled into a single user operation and submitted together, instead of one signature per step.

## The two modes

Account abstraction runs in one of two modes, selected with `--aa-mode`:

| Mode | Flag | What it is |
| --- | --- | --- |
| ERC-4337 | `--aa-mode 4337` | The user operation standard. Actions are bundled into a user operation and sent through an entry point contract. |
| EIP-7702 | `--aa-mode 7702` | Lets a regular EOA act as a smart account for a transaction, so an existing wallet gets account abstraction features without migrating to a new address. |

The paymaster that sponsors gas comes from one of two providers, selected with `--aa-provider`: `alchemy` or `pimlico`.

## How to control it

You do not need to set anything for the default behavior. Account abstraction first with automatic EOA fallback is the default. When you want to force or constrain the path, use the `aomi` CLI:

- `--aa` requires account abstraction and errors if no provider is configured.
- `--eoa` forces a direct EOA signature and skips account abstraction.
- `--aa-provider` picks the paymaster provider (`alchemy` or `pimlico`).
- `--aa-mode` picks the mode (`4337` or `7702`).

See the [Client CLI signing modes](/reference/cli#signing-modes) for the full flag surface, including how `--aa` and `--eoa` interact. For how signing fits into a trade end to end, see [Trade Execution](/trade/execution).
